# Understanding 5.3 Biometrics
Status of this document: 

⚠️ Draft mode.

## EN 301 549 v4.1.0 draft
Normative requirement (EN 301 549 v4.1.0 draft)
> “Where ICT uses biological characteristics, the ICT shall not rely on the use of a biological characteristic as the only means of user identification or for control of the ICT.
> NOTE: Some examples of biological characteristics are fingerprints, eye retinal patterns, voice, and face.”

## Intent
Not all users have the same physical characteristics for relying on biometrics for user identification or for control of the ICT.

This requirement exists to prevent **exclusive dependency on human biological traits for access or control of ICT systems**.
Biometric systems can create barriers when:
- A user cannot provide the required biological input (e.g., missing fingerprint, voice impairment).
- The system fails to recognize valid users due to environmental or technical limitations.
- Temporary conditions (injury, illness) prevent use of the biometric method.
- Users are excluded due to low accuracy for certain populations (e.g., face recognition bias).
The clause ensures **redundancy and inclusivity in authentication and control methods**, so access is not blocked by physical or biological limitations.

### This requirement benefits
- Users with permanent disabilities (e.g., limb differences affecting fingerprints or face recognition).
- Users with temporary impairments (e.g., injuries, surgeries, voice loss).
- Older adults whose biometric traits may change over time.
- Users in environments where biometric capture is unreliable (e.g., gloves, masks, lighting conditions, holding a bag of groceries).
- Users concerned with privacy or unwilling to use biometrics.

### Examples
If you have previously set up identification or control with biological characteristics (e.g., fingerprints) and later have issues with these biological characteristics (e.g., your fingers), you may need to have an alternative way to do the action.

- A smartphone that supports Face ID or fingerprint login must also allow PIN or password login.
- A banking app using voice recognition must also support password or multi-factor authentication that is not based on voice.
- A workplace access system using iris scanning must also provide a badge or PIN-based alternative.

## Evaluation methods
If using the service requires a biological characteristic, the service must also provide an alternative method that does not rely on biological characteristics.
### Failures
- The ICT only accepts biological characteristics as verification methods.
### Passing techniques
- If the function is verified with biological characteristics (e.g., face ID or fingerprint), there are non-biometric alternatives available (such as entering a pass code).
